# V-Kernel TypedCheck Reason Codes

These are the canonical reason codes emitted by V-Kernel `audit_bundle` TypedCheck plugins.

Per the layered shipped-vs-deferred disclosure pattern, each contract listed below is reported in three layers:

- **Contract** — the audit-bundle contract §C1–C16; the spec-level claim.
- **Plugin code wired** — the file in `audit_bundle/plugins/` (and `audit_bundle/discharge/` / `audit_bundle/effect_runtime/`) that emits the reason codes below.
- **Full enforcement scope at v0.2** — see the per-contract notes for named v0.3 deferred items.

**v0.2 status (as of 2026-05-04):** C14 (stamp lattice + signed upgrades), C15 (WASM Component Model effect enforcement + execution-trace attestation), and C16 (refinement-discharge with Z3 verifier-set discipline) are LANDED and enforced under the `default_post_w3_plugin_set()`. Deferred at v0.2: Pyodide / PyScript Python→WASM compile shim; multi-component composition; full WIT-imported `wasi:sockets/wasi:filesystem` toolchain; `locale_bound` enforcement (rejected when combined with `net`/`fs` as `WASM_LOCALE_BOUND_DEFERRED`); Lean/Dafny refinement backends; multi-output `output_idx` wire-format extension. See the audit-bundle contract §§C14–C16 for the per-contract scope.

**v0.3 status (as of 2026-05-20):** C18 (SCITT-statement-shaped release supply chain + plugin-OCI loader + verifier identity tripwire + STH gossip), C19.A (Layer A SCITT-bound counter substrate — reference impl), C19.B (cross-host PeerReview authenticator pairing — reference impl, single-org HMAC only; cross-org COSE_Sign1 reserved for v0.4), C19.C (Layer B RFC 3161 TSA + Roughtime quorum + BLS aggregation — reference impl), and C19.D (key rotation + pre-commitment SCITT log + emergency offline-root) are LANDED. Schema reservations only at v0.3 (no verifier pipeline): C9.1 (append-only attributed file pinning), C14 rigor profile parameterization (substrate deferred to v0.4), C17 attested serving evidence (substrate deferred to v0.4), C20 (semantic fidelity). OF1 (manifest header integrity) closes C14 defense-7 honest-anchor; OF4 ships MANIFEST_SCHEMA.md integrator-facing reference.

> **Sync invariant** — every reason code emitted from a `PluginResult(reason_code=...)` site under `audit_bundle/plugins/` OR raised by a `*Error(ReasonCode.X)` exception under `audit_bundle/extensions/` MUST appear in this file. Cumulative-pre-soak Patch 1 (Gate 1, 2026-05-04) closed an earlier ≥10-code drift between this manifest and the v0.2 plugin code; v0.3 sweep (2026-05-20) closed a ~30-code drift from wave-1 + wave-2 + key-rotation substrate extensions. Future contributors: when adding a new reason code, add the row here in the same patch.

## v0.3 → v0.4 cross-host reason-code migration map

Per the deprecate-with-alias rule (never silent re-mean; never delete a published code). Third-party verifiers pinned to the v0.3 contract use this map:

| v0.3 code | v0.4 disposition |
|---|---|
| `CROSS_HOST_COSE_AUTH_RESERVED_FOR_V0_4` | **DEPRECATED, emit-never alias.** Cross-org COSE_Sign1 is now implemented. A v0.4 cose edge yields `PASS`, or `CROSS_HOST_KEY_NOT_PINNED` / `CROSS_HOST_AUTH_MECHANISM_POLICY_VIOLATION` / `CROSS_HOST_ALG_UNSUPPORTED` / `CROSS_HOST_COSE_MALFORMED` on failure. |
| `CROSS_HOST_AUTH_PROFILE_MISMATCH` | **UNCHANGED meaning** (HMAC declared on a cross_org edge). Still emitted; no re-mean → no versioned successor needed. |
| `SENDER_SIGNATURE_VERIFICATION_FAILED` / `ACK_SIGNATURE_VERIFICATION_FAILED` | **UNCHANGED meaning** ("authenticator failed to verify"); primitive widened from HMAC-only to HMAC ∨ Ed25519/COSE. Not a re-mean. |
| `SENDER_KEY_MATERIAL_UNAVAILABLE` / `RECEIVER_KEY_MATERIAL_UNAVAILABLE` | **RETAINED** for the single-org HMAC path (D3 option-c keeps single-org HMAC). Not deprecated. |
