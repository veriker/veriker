# C18 TUF root — bundled at compile time

> ⚠️ **BOOTSTRAP — NOT PRODUCTION-SIGNED.** Every metadata file in this directory
> is pre-ceremony bootstrap state: `root.json` and `revocation_root.json` carry
> **empty signatures** (the hardware key ceremony has not run), and
> `plugin_allowlist.json` / `sigstore_trust_root.json` carry `TBD-*` placeholder
> digests. This is deliberate and fail-closed — the substrate loads the bootstrap
> root only on the explicit `verify_signature=False` path, and the C18 marker gate
> (`tests/test_synthetic_c18_marker_gate.py`, `REQUIRE_C18_HARDENING=1`) blocks any
> release while these placeholders remain. Production values are filled at the
> v0.3.0-rc1 release-tag-cut ceremony (see **Key ceremony** below). Do **not**
> treat these files as a production trust root.

This directory carries the v-kernel-audit-bundle TUF root metadata files
that are EMBEDDED into the OCI image at compile time (TUF root in CLI binary
at compile time; OOB fingerprints via signed website + DNSSEC TXT + Git repo
on a different CDN; Sigstore BYO-TUF pattern).

## Files in this directory

| File | Owner | Purpose |
|------|-------|---------|
| `root.json` | C18 TUF role | 3-of-3 ed25519 root signers; threshold 2-of-3; ≤90d expiry |
| `sigstore_trust_root.json` | Sigstore-trust-root TUF role (c18-010) | Pinned Fulcio + CTFE + Rekor pubkeys per OQ-2 |
| `plugin_allowlist.json` | Plugin allowlist TUF role (c18-010) | ghcr.io/nexiverify/ plugin OCI digests per CV5 |

## What MUST NEVER appear here

- Private-half key material in ANY form (PEM, raw bytes, SSH-format,
  encrypted PKCS#8). The `load_bundled_root()` defensive check in
  `c18_tuf_client.py` scans for the `BEGIN.*PRIVATE KEY` markers and
  raises `TUFRootKeyPrivateMaterialOnDisk` on any hit. CKP-B audit
  dimension 1.e is HARD-STOP on private material.

- Hex blobs longer than 64 chars that look like private keys without
  PEM headers. (Public-half ed25519 keys are 32 bytes = 64 hex chars
  exactly; anything longer is suspect.)

## Key ceremony — production v0.3.0-rc1

The bundled `root.json` currently in this directory is the **bootstrap**
root — generated deterministically via `release/tuf_root_bootstrap.py`
for the v0.3 audit shape. Production keys are generated by the maintainer at the
v0.3.0-rc1 release-tag-cut ceremony, with the private halves held in
HARDWARE TOKENS (3 separate tokens, threshold 2-of-3):

  1. maintainer-laptop ed25519 (YubiKey FIDO2)
  2. GH-Actions release-bot ed25519 (cloud KMS or sealed secret)
  3. Offline NEXI-controlled ed25519 (cold-storage paper-wallet equivalent)

Rotation: ≤90d expiry. The rotation ceremony pushes a new `root.json` signed
by `threshold` of the OLD keys to attest the NEW keys (TUF "overlapping
signature window" — old+new keys co-sign during rotation).

## OOB bootstrap

The bundled root.json fingerprint is published via three independent OOB
channels per CV1 mitigation:

  1. Signed website at `https://manifest.vkernel.dev/root-fingerprint.txt`
     (signed via Vercel + DNSSEC TXT record at vkernel.dev)
  2. Git repo at `https://github.com/nexiverify/veriker/blob/main/audit_bundle/extensions/_tuf_root/root.json`
     (same file shipped in the release; commit history attests provenance)
  3. DNSSEC TXT record at `_tuf-root.vkernel.dev` (TXT value =
     hex(sha256(root.json content)))

Consumers cross-verify all three at first-trust setup.
